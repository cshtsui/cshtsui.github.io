Forge Extensions

Extensions are libraries of components that are built for use with Forge but are packaged separately.

Core extensions

Core extensions are managed by the Design Systems team and will become core Forge components in the future. A component could be an extension for a number of reasons:

  • It’s new and isn’t stable yet. In this case, it will have a pre-1.0 version number.
  • Integrating it into core Forge would require breaking changes. In this case, it will be added in the next major release.

If an extension author wants to share a core extension before the Design Systems team has a chance to review it, it will be listed as a community extension until it’s reviewed.

Community extensions

Community extensions are extensions built to go beyond what core Forge provides. They can be built by anyone using Forge (the Forge community). A community extension could be separate from core Forge for a number of reasons:

  • It's specific to a certain product or use case.
  • It's large and makes more sense as an entire library on its own.

Community extensions aren’t reviewed or managed by the Design Systems team. Reach out to the owner with any questions or feedback.

Creating extensions

To create an extension, we recommend using the Forge Extension AppFabric generator. It provides everything you need to build, test, demo, document, and publish your new extension. More details and instructions in the generator's README.
