Release Notes

2.1.0 - 05/01/2019

Here's what we've changed since the last release (2.0.0):

New:

  • We've introduced a whole new set of colors - the secondary color palette. Read more about how to use these on the Colors Foundations page.
  • @tesmond has built a Virtualized Table component using Forge! Check it out on the Extensions page under Community Extensions.

Deprecations:

  • The $green-light color variable has been deprecated. That color is a part of our new secondary color palette and is now called $green-secondary-100.

Improvements:

  • DateInput's disabledDays prop now accepts many more types, including functions, objects (containing a days of week array, a from-to range, or before-after range), multiple Moment.utc dates, or any combination of these types. See the new DateInput disabledDays demo or the props table for details. Thanks for suggesting this, @Mike DeWitt!

Maintenance:

  • All stories in Storybook are now wrapped with our <Root> component.
  • The forge-app repo (which is useful for spinning up a new Forge project or making a prototype) is now using Forge 2. Thanks for updating this, @dlynshue!

Bug fixes:

  • Sortable Table headings now align with their columns. Thanks for reporting this, @libby!

Documentation:

  • We've added new Forge Guide demos for Form and ReduxForm showing a comparison between the default and compact layout modes.
  • StatusTag guidance has been improved, with advice on how to properly display Attention- or Critical-level information
  • DateInput's Forge Guide page now includes a new demo showing disabled days.
  • Fixed typos on the Forge Guide ReduxForm page.

2.0.0 - 04/03/2019

Forge 2.0 is now GA!

To see everything we've changed since version 1, read our Forge Upgrade Guide.

Here's what we've changed since the last release (2.0.0-beta.12):

Breaking changes:

  • The deprecated onUpdate method has been removed from SelectionList. Use onChange instead.
  • FormField’s inputWidth prop no longer accepts the value 'large'. Provide a valid CSS width instead. The related class fe_c_form-field__input--large has been removed from the HTML API.

Maintenance:

  • redux and react-redux dependencies changed to peerDependencies in core Forge and in the extensions MultiselectAsync and ToggleSwitch
  • Toast no longer depends on redux, react-redux, or redux-form
  • Core extensions now require Forge version 2.0.0 or newer
  • Test coverage for redux-form and autogenerated icons has been improved

Bug Fixes:

  • Many form elements that were using the muted text color are now using the default text color, including:

    • Unfocused text in Input, DateInput, and Typeahead
    • Multiselect and MultiselectAsync dropdown options
    • RadioGroup and Checkbox descriptions
    • Non-string-matched portions of Typeahead suggestions

  • All form element components (except Textarea) are now the same height

  • All form elements are now vertically centered in Multifield
  • FormError height has been decreased to match most other form components
  • Fixed issues with required Multiselect and MultiselectAsync components not triggering error validation in ReduxForm when left empty
  • Fixed issue where the DateInput calendar was defaulting to the current month and date, even if that date was outside the allowed date range. In those cases, it will now default to January of the nearest valid year
  • ReadOnlyText now uses the input line height (1.25)
  • The padding beneath Textarea is now consistent across browsers
  • Removed right padding on last child of RadioGroup in Multifield (to account for horizontal arrangement)
  • Multiselect and MultiselectAsync’s options dropdown no longer shift when the inputs are hovered

2.0.0-beta.12

  • Typeahead menu placement is now dynamic, and won't overflow offscreen. Also, search substring matches in options are now more visible. See the Typeahead HTML API for new classes.
  • Loader is now accessible. If your Loader does not use the text prop, use the new aria-label prop to indicate loading to a screen reader.
  • Red, green, and blue colors have been updated. They now have WCAG 2.0 AA compliant contrast ratios against white and light gray ($gray-88, rgb(225, 225, 225)) backgrounds.
  • Heading descriptions' line height and top margins have been increased slightly, and they now use font-weight: regular instead of light.
  • We now allow minor version updates of react-select (our previous dependency allowed patches only).
  • Sample "Setting a default option" code for Select has been simplified, and no longer includes a ReduxFormField.
  • Bug fix: ReduxForm will no longer throw a console warning when required fields are invalid.
  • Bug fix: Stepper no longer pushes other page content down when certain steps are selected.
  • Bug fix: Select components no longer shrink in height when used inside of Multifield.
  • Bug fix: Breaklines now show up in Forge Guide code examples.

2.0.0-beta.11

  • Typeahead now announces available suggestions to screen readers. You can overwrite Typeahead's handling of this with the new getSuggestionAnnouncement prop, or provide the text yourself with the new suggestionsAnnouncement prop.
  • Paginator is now more accessible to screen readers and keyboard navigation.
  • The tall layout has been removed from Table. Available values for the layout prop are now default and compact.
  • We no longer depend on a fixed version of highlight.js. This explicit version dependency was added in 2.0.0-beta.10 to address a highlight.js bug.
  • Bug fix: Stepper, RemovableTag and Table sortable header buttons now have explicit type.

2.0.0-beta.10

  • SelectionListItem now automatically uses the text content of its children as the accessible label of the Checkbox. You can overwrite this with SelectionListItem's new aria-label prop.
  • Checkbox now takes an aria-label prop. Use it when rendering a Checkbox without a description to ensure screen reader users know the purpose of the Checkbox.
  • Grid and List icons added to the ForgeIcons iconSet.
  • Updated dependencies with security issues.
  • Bug fix: Accordion header button now has explicit type (Thanks @rvanderheyde !)
  • Bug fix: Checkbox label and Radio Button labels now have the same font-size (Thanks @Alvaro Montoro !)

2.0.0-beta.9

  • DateInput, Heading, SelectionList, Table, and Tooltip are now more accessible to screen reader users.
  • Heading's subheading prop has been renamed to headingDescription.
  • Focus is now trapped in Modals, Overlays, and Lightboxes.
  • KeyboardUserContext added to Root, which notes if users are navigating with a keyboard vs. a mouse.
  • Bug fix: Fixed issue where DateInput didn't process clicks on calendar days in some cases.

2.0.0-beta.8

  • Multiselect is now more accessible to screen reader users. Multiselect options can now be selected by using the Enter key. Multiselect now has improved keyboard navigation. Multiselect now reads correct instructions for selecting options.
  • Multiselect no longer has aria-describedby and openMenuOnClick props. This is due to open issues in react-select that make these props not work.
  • RemovableTag now has a tabIndex prop, which allows setting the tabindex of the button element.

2.0.0-beta.7

  • Multiselect now uses react-select v2, making it more accessible to screen reader users and easier to customize. Multiselect "no options found" message can now be customized with the noOptionsMessage prop.
  • Tabs are now more accessible to screen reader users. Tabs can now take a label to describe all tabs with the tabGroupLabel prop. Users can now use the left/right arrow keys to change tabs, use the Home key to go to the first tab, and use the End key to go to the last tab.
  • Bug fix: MultiField fields now wrap on small screens.
  • Bug fix: MultiField field labels are hidden if none are provided.
  • Bug fix: Hovering over disabled select no longer removes chevron & required bar.

2.0.0-beta.6

  • Accordion now more accessible to screen reader users. Users can tab through AccordionItems, use Enter and Space Bar keys to toggle open AccordionItems, and screen reader will read the headingText of an AccordionItem when it is selected.
  • Card now allows Space for click when onClick handler is provided for accessibility.
  • Stepper now more accessible to screen reader users. Stepper now uses buttons for steps, has updated focus and hover styles, and screen reader will read if step is completed or current.
  • Attention color updated to improve contrast.
  • Bug fix: Accordion chevron now correctly aligned in IE11

2.0.0-beta.5

  • Separated CSS/SASS linting and moved the config files into the forge-gulp-tasks package
  • Added react-a11y eslint plug-in to help with developing with accessibility in mind
  • Made padding-related props more consistent across components. This includes removing padded prop from Accordion, adding a padded prop to AccordionItem, adding a padded prop to Modal, and removing the padding style on the Tabs wrapper component.
  • Updated Accordion styling - the grey background on the content has been removed and default font size has been made smaller

2.0.0-beta.4

  • Changed patient safety alerts to critical alerts. Includes updates to color and typography variables, icons, Banner, and Modal.
  • Restricted icon in Button to only be supported in default size.
  • DateInput auto-completes 2-digit year to 4-digit year (up to 10 years in the future or 90 years in the past)
  • Made input borders lighter.
  • The following layout props can now be set on Form or ReduxForm and will serve as defaults for any rows in the form: compact, labelAlwaysAbove, labelWidth, and nested.
  • Bug fix: Addressed some IE11 form layout bugs.
  • Bug fix: Form submit button now aligns with inputs.
  • Bug fix: Forge styles no longer conflict with resets that set box-sizing: border-box globally.
  • Bug fix: Accordion expand/collapse icon now has title "Collapse" when expanded.

2.0.0-beta.3

  • Enabled importing individual components as @athena/forge/lib/component-name. This can greatly reduce bundle size in some scenarios.
  • Updated Info icon to display i instead of ?.
  • Updated focus styles for WCAG AA accessibility.
  • Various other accessibility fixes for Checkbox, IconButton, and RadioGroup.
  • Enabled adding an icon to Button. API is the same as for IconButton. The icon is not displayed for 'tiny' or 'small' size buttons.
  • Removed all Object.assign usage. Forge should no longer require babel-polyfill.
  • The form's error message display component can now be imported and used as FormError. This could be useful if you're creating a complex custom input and want greater control over when and where error messages appear.
  • Removed all UNSAFE React methods for future version compatibility.
  • Bug fix: RemovableTag removal button size stays constant on small tags (fix by lmcrae)

2.0.0-beta.2

  • Improved accessibility of Button, IconButton, Checkbox and Radio components
  • Bug fix: Updated Table component to pass all style overrides to Rectabular for conditional cell styling (fix by tesmond)

2.0.0-beta.1

  • Modified greys to show up better on low contrast displays
  • Updated dependencies including verified support for React 16.5. Webpack 1 is no longer supported.
  • Changed ReduxMultiField to use redux-form's Fields component to improve performance and code quality. reduxFieldAs prop was replaced with reduxFieldsAs. Further customization is needed for validation to work when setting reduxFieldsAs.
  • The fixedWidth prop has been renamed to inputWidth. This replaces previous inputWidth functionality. The new inputWidth prop is now supported by all input components (Checkbox, DateInput, Select...)
  • Bug fix: Radio extra props will be passed to input instead of container

2.0.0-alpha.4

  • Added MultiField, ReduxMultiField, RemovableTag, and StatusTag components
  • Compact and labelAlwaysAbove props added to FormField
  • React 16.5 removed from peer dependency range to avoid a suspected issue. Investigation ongoing.
  • Select and Tabs triangles have been replaced with the Expand icon
  • Bug fix: Typeahead no longer causes horizontal scrollbar to appear unnecessarily
  • Bug fix: DateInput's calendar month and year selection works in Firefox when used in a Modal or Lightbox
  • Bug fix: Loader spinner animation is now smooth in Chrome

2.0.0-alpha.3

  • Migrated Multiselect, Typeahead, Paginator, Lightbox, Card, Input and Stepper from Forge Extensions to Forge Core
  • TextInput component has been removed and replaced with Input component
  • Components that take an icon prop (IconButton, Input, Typeahead) can now take an iconSet prop
  • Added Upgrade Guide from Forge 1.x to Forge 2.0
  • Bug fix: Tabs no longer collapse under all widths under certain conditions
  • Bug fix: Static methods of DateInput (e.g. DateInput.formatDate) available again
  • Bug fix: DateInput calendar now closes on scrolling

2.0.0-alpha.2

  • Added Root component (and underlying ThemeProvider and ThemeConsumer HOC) to support providing a theme to Forge components
  • Updated DateInput, Modal, and Tooltip to take rootClasses from theme
  • Bug fix: ReduxForm no longer throws a warning when using onSubmitSucceeded

2.0.0-alpha.1

This version has been branched from 1.8.0 and includes the following changes:

  • Bug fix: ReduxForm no longer throws error on form submit when fields require validation
  • .fe_f_inherits styling is now included in .fe_f_all
  • Tabs default padded prop is now true
  • Radios without a RadioGroup have a styling change. display is now inline-block to match checkboxes
