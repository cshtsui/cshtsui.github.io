Release Notes

1.5.0

  • Expanded access to Reactabular API through Table. Added props include onRow and columns.props among many others.
  • FormField provides mixins for input width and required bar for use by custom input components
  • Bug fix - FormField was incorrectly setting width of nested inputs
  • Updated CI to deploy workspace to S3

1.4.0

  • Added expanded and defaultExpanded props to AccordionItem
  • Bug fix - Modal content is now visible even on very small viewports
  • Bug fix - Restored link styles
  • Added snapshot tests for some components

1.3.1

  • Bug fix - Checkbox and Radio more visible on dark backgrounds
  • Bug fix - Accordion padding is now distributed properly
  • Bug fix - fe_f_inherit no longer changes Label cursor

1.3.0

  • Added sortable columns in Table component.
  • Accordion component - Published documentation. It’s now fully supported.
  • SelectionList component - Published documentation. It’s now fully supported, including an API for use in ReduxForm.
  • Tabs component - Published documentation. It’s now fully supported.
  • Tweaked required bar alignment for Checkbox and RadioGroup in FormField
  • Tested with React 16.3 and expanded peer dependency
  • Added ability for custom inputs to define their own required field validator as Component.validateRequired
  • Added a changelog file
  • Now fail linting on warnings in addition to errors
  • Moved Box component to forge-extension repository
  • Bug fix - Tooltip in Modal not visible.

1.2.0

  • Added styling for typographic list elements (ul, ol) and fe_u_list-style--no-bullet utility class.
  • List component - Published documentation. It’s now fully supported.
  • Banner component - Published documentation. It’s now fully supported.
  • Accordion component - Added, not yet documented or fully supported. Thanks @misteele and @cbarringer!
  • Bug fix - More robust handling of font-family inherited CSS property to prevent accidental overriding.
  • Added fe_f_inherits for more robust handling of numerous inherited CSS properties.
    • Will be bundled into fe_f_all in 2.0, but left separate for backwards compatibility.
  • Bug fix - Modal has extra bottom padding when excluding buttons. Thanks @kymurray!
  • Added the $base-prefix Sass variable for customizing the base selector (and therefore specificity) of all Forge component styles.
    • If you customize the prefix, make sure it will apply to both your standard Forge root and to any portals (Modal, Tooltip, DateInput picker) that you create.

1.1.0

  • Bug Fixes
    • Fixed issues with DateInput around positioning, validation, shortcuts, styles Breaking change if you've developed your own component with internal state validation -- this is not the same as using constraints (UXDS-1261, UXDS-1300, UXDS-1305, UXDS-1306, UXDS-1310).
    • Fixed Checkbox checked not set by ReduxForm initialValues (UXDS-1380). Thanks @kymurray!
    • Fixed Table row hover in Safari (UXDS-1307).
    • Fixed Select border color (UXDS-1303).
    • Fixed too large margin on ReadOnlyInput (UXDS-1312).
  • Added Banner component, currently undocumented. Thanks @znielsen!
  • Now passes npm-kit-validate.
  • Added example starter component (not exported) called Box -- use this as a blueprint to begin any component contributions or extensions.
