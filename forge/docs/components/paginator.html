Usage

Pagination is the process of dividing content into discrete pages. The Paginator component provides a tool to navigate among those pages. It does not break up the content itself; that must be implemented separately.

This component shows clickable page numbers, with the current page number highlighted. Arrows on the left and right navigate to the previous and next pages. An arrow is disabled (grayed out) when there is no previous or next page.

Paginator is a good fit when you have a set number of pages to navigate. Paginator does also include an open-ended option for navigating unlimited pages. This is helpful if you don’t know the full number of pages on first load.

Infinite scrolling (loading content continuously as the user scrolls down) is an alternative to pagination. Paginator is not suitable for this.

Do not use Paginator to link to other separate pages.

Variations

Paginator has several variations that require progressively more space but offer additional functionality:

  • Compact - Only displays the current page, plus arrows for navigation.
  • Default - Adds links to the first few, last few, previous, and next pages.
  • Jump - Adds an input where the user can specify which page to jump to.

Design

Layout

Paginator is typically right-aligned at the top of the content page/block.

For workflows where users are likely to scroll from top to bottom and then immediately go to the next page, Paginator can be included at the bottom of the content page instead.

If the workflow involves scrolling up and down the page, make Paginator sticky (fixed) for easier access. If making it sticky isn’t possible, then consider including Paginator above and below the content.

Page numbers

Page numbers can be only numbers, not other characters or icons. This cannot be customized currently. In the default and jump variations, how the pages are displayed depends on the total number of pages.

  • Seven or fewer pages - links for all of the pages are displayed.
  • More than seven pages - some pages aren't displayed and are replaced by ellipses. The first, last, previous, and next pages are always displayed.
  • Unknown or infinite pages - an ellipsis appears after the page links to indicate that there are more pages.

Implementation

Paginator only provides the navigation UI to support pagination. It doesn't implement the pagination logic. That's left up to the developer.

Note that selectedIndex is zero-indexed, while pages are labeled beginning with "1". That means selectedIndex={0} will show a highlighted "1", selectedIndex={1} will show a highlighted "2", and so on.
