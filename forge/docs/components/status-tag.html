StatusTag

StatusTag is a visual indicator of status, category, or another aspect of an object. Use it for calling out non-urgent information.

Usage

StatusTag is a visual marker with very short text inside it. Place it next to an object to call out an aspect, to label the object, or to provide useful information, such as:

  • Status
  • Category
  • Importance

Adding tags can help users to scan a page and quickly assess what’s on it and what needs action.

StatusTag is intended for displaying non-critical, low-urgency information. It should not be used for calling out medium- or high-urgency information. Use the Attention or Critical icons and styling in those cases.

StatusTag isn’t interactive. For a tag that can be dismissed by users, consider using RemovableTag instead. This tag represents user input and is commonly used for searching and filtering content.

Design

The tag’s width depends on the length of the label text inside it. Labels should be short (1-3 words) and displayed on one row with no wrapping. Labels are text only (no icons).

StatusTag automatically displays label text in all caps. This helps differentiate it from other elements on the page.

StatusTag uses inverted colors (white text on dark gray background) to stand out and provide sufficient contrast for its text. Other colors can be used for the tag fill if necessary. Please consult with the Forge team when using other fill colors. The color should depend on the type of information being communicated and align with Forge color semantics whenever possible.

Implementation

Enter label text for StatusTag in title case (e.g., “In Progress”), not in all caps. This supports assistive devices such as screen readers, which may have trouble reading out all-caps text. The all-caps display is handled automatically.
