Banner

A Banner is a temporary message or alert informing the user of the status of a given page.

Usage

Page level banners are used to inform the user of an occurrence that has an impact to the entire page. If a more specific level of message/alert can be used it should be used (e.g., section level pertaining to the section, inline error on a form field, etc.).

Alert Types

Banners normally have a color associated with the type of message they are trying to convey. Different colors should have different behaviors.

Purple - info

A purple banner is informational, giving general tips and helpful information.

Green - success

A green banner is to convey a confirmation or success message.

Yellow - attention

A yellow banner normally is conveying an error or alert and needs some form of action to be taken. The user has to dismiss the banner before it disappears ensuring that user acknowledge it was there.

Red - critical

A red banner is of extremely high importance. It should be used only for patient safety and other critical issues (such as business issues that affect the financial health of the client). It absolutely requires an action and should not disappear until action has been taken. The banner does not prevent users from taking other actions. If action needs to be blocked, the Modal should be used instead.

Design

A page level banner should sit below the Global Navigation at the top of a page and header, but not above any left hand side bar. The banner resizes fluidly with the page body area.

Banner header text should use title case (e.g., "Security Update", "Problem with Data"). Banner content can include text links. Banner content should not include icons, images, or interactive components.

Implementation

Banner children should always be implemented using one or more BannerItem components. This ensures proper layout and spacing. All banner features such as the alert, dismissible icon, and heading text are customizable by changing props on the Banner or BannerItem components.

Animating a Dismissible Banner

Animation for showing and hiding the banner does not ship with Forge. If you are looking to build this type of functionality we recommend using the react-transition-group library. You can find an example of its usage in the demo section.

Banner Props

Prop Type Default Description
alertType custom: ForgePropTypes.alertTypes 'info' Gives a default left border and header with icon that corresponds to status level
className string Adds a class to the root element of the component
onHide func Callback called when close button is clicked, if not included no close is shown
...rest Passthrough props to the root div element
