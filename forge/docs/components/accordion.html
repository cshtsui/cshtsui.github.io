            compliant cryptographic hash function is used, which allows us to confirm a user’s password without
            persisting it in our system. Allowed passwords can be configured by practice security administrators, but
            we discourage changing parameters from the defaults. These parameters include the minimum password length,
            the minimum password complexity, the number of inactive minutes before session timeout, the number of
            failed login attempts before the account is blocked, and the number of unique passwords which must be
            used before a password can be reused.</p>

Usage

Use Accordion when there is long, related content that the user may only wish to see part(s) of at a time. Accordion helps reduce overall page length / scroll by representing each piece of content as a summary title. The user can choose to reveal full details by clicking on each Accordion section header.

If the complete content presented in Accordion is very long and / or makes up most of the content presented in the page, consider using the Tabs component instead.

Design

Once expanded, an Accordion section remains expanded unless a user explicitly closes that section; this enables users to compare content across sections of an Accordion. Accordion sections animate smoothly when expanding and collapsing to reinforce continuity.

Accordion and its contents should take up the full width of their parent container and flow responsively when that container resizes. Content includes padding by default; set the AccordionItem prop padding to false to let content span its full width. Avoid nesting content too many levels down and putting large or long components like data-dense tables within an Accordion.

Item headers

Be as concise as possible in the heading text for each Accordion section.

The header of each Accordion item can include minimal custom content such as an icon. Use it to convey status or other high level information. The custom content shouldn't overwhelm the heading text and is limited to no more than one third the width of the header.

Accessibility

  • Keyboard-only navigation
    • Users can navigate to accordion headers with the tab key and then expand (enter key) and collapse (space key) the selected accordion pane.
  • Screen reader support
    • When an accordion header is selected, screen readers will read the header text and describe the content section as expanded or collapsed.

Implementation

Accordion content should always be implemented by using one or more AccordionItem child components.

Each AccordionItem tracks its own expanded state. To make an item expanded on initial render, set defaultExpanded. To explicitly control when it expands and collapses, set expanded and listen to onExpandedChange.

By default, the AccordionItem children will not be rendered (i.e., instantiated and rendered) while the AccordionItem is collapsed. This helps keep the initial render of Accordion quick. Set mountedWhileHidden to true if it’s important for an AccordionItem child component to be available even while its AccordionItem is collapsed.

It is a best practice to keep the headingTag value the same (e.g., all 'h3') among all AccordionItems belonging to a single Accordion, as all AccordionItem components are at the same level of the heading hierarchy.

Accordion implements the List component under the hood to generate appropriate semantic HTML. Similarly, AccordionItem implements ListItem.
