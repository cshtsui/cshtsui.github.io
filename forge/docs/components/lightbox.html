Lightbox

Lightbox is a large pop-up. Use it for presenting a workflow without needing to leave the current page.

Usage

Lightbox is used to perform a sub-workflow that is part of a larger workflow. It disables the main content until the user either completes the workflow or aborts it by closing the lightbox (using a button or by clicking in the background overlay).

Some typical use cases include:

  • Launching a sub-workflow needed to fill in a single field in a larger form (e.g., searching for a local laboratory to send an order to).
  • Presenting information that has to be kept explicitly distinct from the main page to avoid confusion (e.g., viewing a spouse’s medical information during a fertility treatment visit).
  • Presenting the user with content they need to review before continuing. This could be either Modal or Lightbox depending on the amount and nature of the content. Modal is intended for short, alert-style interactions.

Use Lightbox sparingly. In terms of interrupting a user’s main workflow, it’s a compromise between embedding content within the workflow and navigating away from it. Like many compromises, it’s an easy solution but rarely the ideal one.

Some red flags to avoid:

  • Requiring the user to repeatedly open and close lightboxes as they go through a workflow.
    • Consider using a side pane or other persistent solution instead.
  • Navigation within a lightbox, such as tabs or a menu.
    • If the sub-workflow is this complex, better to navigate to a new page and use breadcrumbs to make it easy to return.
  • Heavy workflows where all the user’s input could be lost if the lightbox gets accidentally closed.
    • Consider presenting the workflow on a new page instead.
    • Lightbox can have its default close/dismiss options disabled (see Design), but this should rarely be done.
  • Lightboxes that open other lightboxes or modals.
    • Do not do this.

Design

Lightbox covers the main page with a darkened overlay, disables the main page’s content, and displays a pop-up/dialog. The pop-up has three possible widths: medium (the default - 600px), large (800px), and full (90% of the window).

The pop-up contains three sections of content: header, body, and footer.

Overlay

By default, Lightbox is dismissed by clicking anywhere in the overlay (the darkened background outside the pop-up). This behavior can be disabled when presenting critical information or a required workflow, but it should be done sparingly.

Header

The header includes a title and close button. The title should be short and provide context to the lightbox’s contents. Title text should use title case. The close button can be omitted if you want to prevent users from closing the lightbox without completing the workflow.

Up to two IconButtons can also be included in the header. These should be obvious secondary actions with brief descriptions that don’t require explanation (e.g., Print File, Email). They should not complete the workflow. If adding IconButtons, be sure to test your layout on the smallest screen size you support.

Body

The body contains the content or workflow presented by the lightbox.

Footer

The Lightbox default footer contains a single primary call-to-action Button that completes the workflow. This action closes the pop-up and returns the user to the main page. The footer is optional but recommended.

If you need to add more buttons to the footer or customize it in other ways, you will need to create your own custom “footer” inside the body section instead. Any additional buttons should be secondary.

Button text should use title case.

Implementation

Closing Lightbox

Lightbox does not close itself. Its parent is responsible for setting the show prop to false in the onHide and onPrimaryClick callback functions.

onPrimaryClick is called when the primary button is clicked. onHide is called for all other interactions that should close Lightbox (e.g., clicking on the overlay). Therefore, implementing the close functionality within onHide is required, unless disableClose is set to true. If disableClose is set to true, the header’s close button is not displayed and the lightbox will not be closed automatically by clicking in the overlay.

Footer

Currently, the footer comes with one single button that can be customized with primaryButtonText and onPrimaryClick. If you need multiple buttons or other customizations, you’ll need to implement a footer within the body.
