Usage

Often used to supply options for a field that is part of a Form, the Select is useful for enforcing structured data conventions by eliminating variability in user input.

Number of Options

Use a Select when the collection of options exceeds a reasonable amount to show as permanently visible choices in a RadioGroup. Present users with their choices upfront rather than in menus needing an interaction to see selection choices. Consider using Select for collections with six or more items.

For truly massive collections (e.g., list of all providers in a large hospital), consider a design strategy that reduces the number of options through filtering or information taxonomy changes. Multiselect and Typeahead provide filtering for long lists of options.

Design

The Select component implements the native HTML <select> element with visuals that are largely handled by the operating system. This means that CSS, and hence Forge, have limited effect on <select> input styles, especially for the contents of the pop-up menu. The Forge Select visual design considers these limitations and is designed for cross-browser/operating system consistency.

The use of this standard HTML <select> is encouraged when choosing from a reasonable collection of items because:

  1. It is performant across all browsers and its behavior is standard to users of a particular operating system.
  2. The native implementations of <select> in mobile browsers provides superior touch input experiences and is familiar to users.
  3. The pop-up portion of the <select> smartly positions itself relative to its location on the page, including extending out of the browser window if necessary. The <select> guarantees that the the menu content will be viewable.

Form field label text should use sentence case (e.g., "Relationship to patient").

Find more guidance on the Form Design Foundations page.

Default Select Option

Select defaults to having a placeholder option labeled "- Select -" if none of the options in the menu is designated as the default choice. The placeholder option can be removed with the hasNullDefault prop.

Implementation

Updating Select Options

If the options in a Select need updating after its original render, be sure to instantiate a new array and re-assign it to the options prop. Merely updating the existing array will not trigger a re-render of the component with the options in a modified array.

Setting a Default Select Option

Set the hasNullDefault prop to false. The first option in the menu then becomes the default and the placeholder option is removed. To further refine the default choice, set the defaultValue prop to one of the option values:

<Select
    hasNullDefault={false}
    defaultValue={20}
    options={[
        { text: "A", value: 10 },
        { text: "B", value: 20 },
        { text: "C", value: 30, disabled: true }
    ]}
/>

Select Props

Prop Type Default Description
children React option elements Select options can be set through the options prop or through children, not both
className string Adds a class to the root element of the component
disabled bool false Impacts the styling of the select in addition to preventing selection
error string Highlights the input when defined (specific string doesn't matter)
hasNullDefault bool true Specifies whether to include an explicit null option. Only unset this if there is a good default option
options custom: ForgePropTypes.selectOptions If an option is supplied as a primitive, it's both the value and the text of the option
placeholder string '- Select -' Text shown for the null option. Read the UX Content Guidelines before changing
...rest Passthrough props to the <select> element
