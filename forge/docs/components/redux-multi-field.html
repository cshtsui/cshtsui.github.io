ReduxMultiField

ReduxMultiField wraps the inputs of a MultiField in Redux-Form Field components to enable easy validation and storage.

Usage

ReduxMultiField provides the state management of Redux Form's Field for each of the inputs in a MultiField, as well as built-in validation via the required and constraints props. It otherwise has the same API and pass-through behaviors as MultiField.

Design

The design documentation for MultiField applies to ReduxMultiField.

Find more guidance on the Form Design Foundations page.

Implementation

ReduxMultiField is an extension of MultiField that provides validation logic and connects the fields to a redux store using the redux-form library. It is analogous to ReduxFormField and how it extends FormField.

ReduxMultiField has a fields prop similar to MultiField that takes an array of props for each input. It adds support for the constraints and showErrorsBeforeTouched props in fields. It also adds reduxFieldAs as a top level prop (not within fields). See ReduxFormField for more details about these props and how theyre used.

ReduxMultiField is meant to house separate but related inputs, so each input is stored and validated separately. If you want to combine the values and validation for multiple inputs, e.g., a telephone input with a separate input element for the area code, you can create a custom input component (see the Input component API section of the Forge Extension Generator Readme for more info).
