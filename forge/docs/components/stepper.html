Stepper

Stepper is a visual guide indicating where a user is in a workflow. It may also be used for navigating the workflow.

Usage

Stepper shows progress through a workflow of several steps. It lists all steps, highlights the user’s selected (current) step, and shows which steps have been completed.

Stepper is recommended for workflows with 3-7 steps. It is only a visual indicator and navigation tool. It does not handle advancing the workflow or provide any application state management. This must be implemented separately as part of application logic. Stepper exists only to reflect progress in a workflow.

Stepper can allow navigating to steps out of sequence, letting users go back to completed steps or skip ahead in the workflow. Not all workflows can - or should - support this. Before supporting this functionality please consider:

  • Can the workflow be done out of sequence, or does each step build on the previous steps?
  • Are there technical or user experience constraints around users entering data out of order or going back and editing data they have already entered?
  • Could letting the user skip around in the workflow cause confusion or error?

Variations

  • Default
    • Displays steps plus checkmarks on completed steps
    • (optional) Displays numbers on selected step and incomplete steps
    • (optional) Displays labels below steps
  • Compact
    • Displays steps only (no numbers, labels, or checkmarks)
    • Automatically switches to this version in small viewports
  • Navigable
    • Adds links to steps and labels

Design

Each workflow page should include a heading separate from the Stepper component, preferably above the Stepper. Do not use the labels in Stepper as a page heading; labels are not visible in the compact version, which automatically switches on in small viewports.

Stepper is typically center-aligned above workflow content. The horizontal lines between steps expand or contract based on viewport width. Currently, there is no vertical layout, only the horizontal default.

Stepper can display step numbers and labels. Both numbers and labels are optional but recommended. Displaying them helps users see what work they have already completed (important for people with memory issues or people who may be interrupted mid-task) and anticipate next steps. They are also useful for instructional text and training, making it easy to refer to “Step 2” or “Step 3”. Label text should use title case.

Implementation

By default, Stepper marks previous steps as completed, but this can be controlled explicitly instead with completedSteps.

All steps globally or each step independently can be defined as navigable by setting navigableSteps. Each navigable step (with label, if displayed) is wrapped in a link.

The selected step is never considered navigable or completed, regardless of the values of those props.

Stepper Props

Prop Type Default Description
selected (required) number The current step.
steps (required) oneOf
  • number
  • arrayOf string
 The set of steps defined as a number or array of strings if labels are desired. Can only be a number in compact mode because labels aren't shown.
className string Adds a class to the root element of the component.
compact bool false Component will display in its compact form.
completedSteps arrayOf bool Determines which steps are complete (i.e., which steps have checkmarks)
navigableSteps oneOf
  • bool
  • arrayOf bool
 false Defines steps that are interactive. Can be a boolean array or a single boolean for all steps.
numerical bool true When true steps will be labeled with step count.
onSelect func Triggered by clicking a navigable step. Step navigability is defined by the navigableSteps prop.
...rest Additional props are passed to the component's root element.
