Usage

Use Tabs when the page has related groups of content that the user only needs to see one at a time. Tabs enable the user to switch between the multiple views quickly without changing the context of the page.

Variations

Horizontal Tabs

Horizontal Tabs are usually placed at or near the top of a page. The default layout of Tabs places each additional tab to the right of the previous one.

Collapsed Tabs

The Tabs labels collapse into a single Select menu when there is no longer room for all the Tab labels horizontally.

Design

Keep the name of each tab to no more than two words and as few characters as possible. Labels should be in title case.

Accessibility

Keyboard-only navigation

When the current tab is focused:

  • Left arrow key: Switch to tab to the left.
  • Right arrow key: Switch to tab to the right.
  • Home key: Switch to first tab.
  • End key: Switch to last tab.

Screen reader support

  • When the Tabs component receives focus, the screen reader will announce the Tab title and its position, then the tab group label (if it is provided).
    • e.g., "Payment, selected, tab, 1 of 4, Check Out, tab group"
  • When the selected Tab changes, the new tab title and position will be announced.

Implementation

Tabs content should always be implemented by using one or more TabPane child components.

Selected Index

The selectedIndex prop only determines the initially open TabPane and will be overridden by users clicking on other tabs. Further changing the selectedIndex prop to change the open tab after the component has been mounted will not work. For programmatic control over which tab is open, use the underlying TabsControlled component instead. TabsControlled will not manage user interactions on the tabs by default, but provides an event handler, onTabsChange, for implementing the desired functionality.

Automatic Collapsing

Tab labels switch from a horizontal layout to a Select when the sum of the minimum width of all labels exceeds the available width of the Tabs parent element. This will update on browser resize, but not on layout changes within the page. Layout changes in the page that impact the tab labels layout require triggering a resize event manually. This should be done sparingly, because the event is global.
