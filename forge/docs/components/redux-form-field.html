ReduxFormField

ReduxFormField wraps a FormField in Redux Form's Field component to connect each individual input to the application's Redux store.

Usage

ReduxFormField provides the state management of Redux Form's Field, as well as built-in validation via the required and constraints props. It otherwise has the same API and pass-through behaviors as FormField.

Design

The design documentation for FormField applies to ReduxFormField.

Find more guidance on the Form Design Foundations page.

Implementation

The implementation guidance for FormField also applies to ReduxFormField.

Validation

ReduxFormField has several types of validation behavior available:

  • onblur: This is the default validation behavior. When focus leaves a form field, that field is validated. If there is an error the inline error notification appears right away.
  • onsubmit: When the form is submitted for server-side validation. An example of setting up submit validation can be found in the Redux Form documentation.
  • asynchronous: The form checks validation against the server in real time as data is being entered. This might be determining username availability or comparing certain types of codes. An example of setting up asynchronous validation can be found in the Redux Form documentation.

Simple validation such as required fields can be handled at the individual ReduxFormField level. ReduxFormField uses validate.js to make validation easy. ReduxFormFields take a constraints prop — a validation constraints object which might look like this:

{
  email: {
    message: 'A valid email is needed in case of emergencies'
  }
}

This structure is similar to the constraints passed to validate.single.

More complex validation, such as testing that two password fields are equal, should be done by the form or another parent of all the inputs involved. The parent should then set the error prop of ReduxFormField.

Creating a custom input component for use with ReduxFormField

Forge provides an example starter repo for doing just this. (This repo is still a work-in-progress, so feedback is welcome.) The source code of the RadioGroup (simpler) and DateInput (complex) Forge components also serve as examples of how to create a custom input type for use with ReduxFormField. Consult the documentation in the Redux Form library around the requirements of the Field component, which is what Forge uses to connect the input field to the Redux Form library.
