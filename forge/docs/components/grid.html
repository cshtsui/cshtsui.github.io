Usage

The grid in this guide actually refers to the use of GridCol and GridRow components. The basic description of what a flexible grid does and its structure overall is covered in the grid foundation topic.

Vertical Alignment

Content should be top-aligned within a column. However, in the case where content of different heights creates a displeasing layout, the content within a column can be center-aligned vertically.

Nesting

Users can nest grids within another GridCol or GridRow. The gutters of nested grids hold the same width as gutters in the parent grid; a doubled edge gutter can be avoided by using the nested prop. Use nesting with care, as deeply nested grids (i.e. > 2 levels) can yield some undesirable results and be difficult to troubleshoot across breakpoints.

Stacking

Users can order GridCol components within a GridRow vertically according to their order of appearance in GridRow by default. The most common case for this is on a small breakpoint for mobile devices. However, this stacking behavior can be extended to the medium and large breakpoints via the stackUntilMedium and stackUntilLarge properties.

Ordering

The GridCol components within a GridRow are ordered according to their order of appearance in GridRow by default. Users can change this using the order prop in GridCol. Changes to order can help tweak the positioning of elements when designing highly fluid user interfaces, and can be useful in conjunction with stacking.

Design

The best practice is to first design for the smallest screen size your application supports and then plan how the layout can adapt to better utilize the extra space on larger screens.

The grid takes getting used to for those with no prior experience with responsive layouts or those who have to-date relied on manual percentage declarations and custom alignment of elements. Here are some key things to remember:

  • Grid is intimately tied to breakpoints and, hence, the CSS media query system. Always think in terms of how the layout should appear at different breakpoints.
  • Trying to achieve pixel-perfect designs with the grid leads to frustration. Think fluidly. Plan for the range of viewport widths within each breakpoint, as well as between.
  • If the user interface using grid will be displayed in a container that has a width significantly smaller than the browser viewport (e.g. a dashboard widget), the breakpoint system may not be useful. Grid can still be used, but be aware that the breakpoint system is keyed to the browser viewport.
  • Use as few breakpoint declarations in columns as possible. The more breakpoints, the more adjustments that need to be tweaked. In many cases, just defining small and medium layout policies are enough.
  • Grids can be nested, but in practice use cases involving more than 2 levels of grids is rare. The deeper the nesting, the more difficult it becomes to debug problems.

Use in Forms

The grid is the best way to quickly achieve responsive forms. The FormField component uses grid to achieve its responsive layout and represents the method of creating a canonical Forge form.

Most form content should be laid out using the grid. The grid is very capable and offers significant savings in development time versus coding with raw CSS flexbox rules for any kind of user interface that needs to be viable at multiple viewport widths.

Overall Structure vs. Fine Positioning

The grid is most useful for achieving fluid, medium- to coarse-grain layout policies that can be clumsy to handle consistently using traditional CSS methods. Conversely, fine positioning at the small scale might better be handled using regular browser flow, flexbox, explicit positioning, floats, spacer elements, padding/margin offsets, etc.

Go with the [Normal] Flow

A good rule of thumb is that if the content can be presented using the normal rendering flow of the browser, it should be fine to use with the grid. This should cover the majority of cases, because designs should be maximizing the use of the normal flow.

Elements requiring relative, absolute, or fixed positioning -- rules that take layout control away from GridCol or GridRow -- will likely yield unsatisfactory results.

Implementation

The grid is made up of GridRow components whose children are GridCol components. These React components are convenience wrappers that generate elements with the appropriate CSS classes. The CSS does all the work and is based on Foundation's Flex Grid.

Column widths and offsets are specified in units. Each unit is 1/12 the width of the row, minus the gutters at the edges of the column.

As described above, the layout can be customized for each breakpoint of: small, medium, and large. When a prop is set or class added for a certain breakpoint, it also applies to all larger breakpoints until another is explicitly set.

For example, if a GridCol has width={ { small: 12, large: 6 } }, it gets a width of 12 units (full row) on small and medium screens and a width of 6 units (half row) on large screens.

Vertical alignment is determined using the align props of the GridRow (for all columns) or the GridCol (for a single column). The possible alignments are 'top', 'bottom', 'middle', and 'stretch'. These are based on flexbox align-items (or align-self) and correspond to 'flex-start', 'flex-end', 'center', and 'stretch', respectively.

GridRow Props

Prop Type Default Description
align custom: GridCol.ALIGNMENTS_PROP_TYPE Vertical alignment of all GridCols in GridRow; stretch is the default behavior
className string Adds a class to the root element of the component
columns shape
  • small - number
  • medium - number
  • large - number
 Number of columns in the GridRow by breakpoint; overrides width set by child GridCol and uses auto sizing
nested bool Whether the GridRow is nested in a GridCol; sets negative margins to overlap parent gutter padding
removeGuttersLarge bool Remove gutter padding from the large breakpoint width for all GridCols in the GridRow
removeGuttersMedium bool Remove gutter padding from the medium breakpoint width and larger for all GridCols in the GridRow
removeGuttersSmall bool Remove gutter padding from the small breakpoint width and larger for all GridCols in the GridRow
restoreGuttersLarge bool Restore removed gutter padding from large breakpoint width for all GridCols in the GridRow
restoreGuttersMedium bool Restore removed gutter padding from medium breakpoint width and larger for all GridCols in the GridRow
stackUntilLarge bool Stack all GridCols in the GridRow until the large breakpoint; then restore default auto sizing
stackUntilMedium bool Stack all GridCols in the GridRow until the medium breakpoint; then restore default auto sizing
...rest Passthrough props are added to the div element
