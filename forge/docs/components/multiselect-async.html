MultiselectAsync

This variation on Multiselect lets users choose multiple options from a dropdown list. The option list can be retrieved asynchronously.

Forge Extension

You must import this component separately.
It is not part of Forge Core yet. It may undergo further design/API changes. See the Extensions page for more info.

Usage

MultiselectAsync is a variation on Multiselect, a selector that allows users to choose multiple options from a dropdown list by typing or by checking off options. It is compatible only with Forge 2.0.0-beta.8 and above.

The main differences in this component are:

  • Asynchronous fetching of the dropdown options
  • Display of a loading state when option filtering begins
  • No built-in option filtering (including string matching) based on user input

When the user starts typing, the loading state is displayed instantly, followed by the dropdown list of options when they are loaded.

Unlike in Multiselect, simply clicking into or focusing on the input field does not reveal the options by default. This behavior can be reset so that the options are revealed, or it can be customized so a predefined set of options is revealed instead. For example, if your full options list spanned hundreds of medications, you could provide a short "top 5" list for users to choose from without having to type and filter.

MultiselectAsync does not automatically handle option filtering (including string matching) based on user input. You must build this functionality into the loadOptions function, which handles fetching option data.

Design

Refer to Multiselect for detailed design guidance.

Loading State

The loading state displays a text string. Its default value is "Loading …". This can be customized if there are concerns about clarity or usability (e.g., "Loading procedure codes …", "Loading diagnoses …").

Accessibility

Screen Reader Support

Known bug: The loading state text does not get read aloud. The screen reader says only that there are zero options until the options are finally loaded. See Jira issue UXDS-2326 for the status of this problem.

Implementation

MultiselectAsync can be used on its own or within a FormField or ReduxFormField for features like field labels and error messaging.

Under the hood, this component extends the Multiselect Forge core component, which relies on the React Select v2 library. The underlying HTML elements that are rendered include a text input and a select component.

Option Filtering

Unlike Multiselect, MultiselectAsync does not automatically handle option filtering (i.e., matching the user input string to options). This must be built into the loadOptions function. For examples of string matching formats to consider, refer to the Multiselect guidance.

Controlled vs. Uncontrolled

To use MultiselectAsync as a controlled component, set the value prop. To set a default value but allow it to control its own state, use the defaultValue prop instead.

Note that that what these props accept is different than what the same props accept in the non-async Multiselect Forge component.

Load Options on Input vs. Load Options on Mount

By default, MultiselectAsync will not load options until the user begins typing in the input field. If you would like to display options before user input (i.e., on click or focus), you can do so using the defaultOptions prop.

You have the option of providing defaultOptions with a predefined set of options to display or setting defaultOptions to true, which tells MultiselectAsync to load options on mount using the function provided to the loadOptions prop.

Caching

MultiselectAsync sets the cacheOptions prop to true by default. This causes the component to cache the options that are loaded for each unique input. Set this prop to false to disable this caching behavior. Be mindful of the size of your cache to maintain good application performance. cacheOptions will continue to cache loaded options until the prop's value changes; it can be manually reset or altered at any time.
