Usage

Card is meant for a short summary of information with relevant actions below. Text content should be succinct and not exceed a single paragraph. Media content should be limited to a single large piece of content (image, video, etc.), and it should always be full width at the top of the card. Action buttons associated with the content should be placed at the bottom of the card.

A single Card should represent a single thing. If you want to represent multiple things, you should use multiple cards or a different pattern, not a card with a list in it. If you want to represent more complex structure, use a different pattern. For example, don't put a Card within another Card.

Interactive Cards

Card can be used as a summary, in which case it often needs to provide access to full details elsewhere. To provide this access, the entire card can function as a link. When doing this, avoid including other interactive elements within the card body; the user could accidentally trigger the card when interacting with them. Links and buttons can be safely added to the button row at the bottom of the card.

Design

Structure

Card consists of:

  • Media slot - Space at the top for an image, sparkline, or other small graphic element.
  • Heading - Concise description of what the card represents. Heading text should use title case. You can also include a heading description.
  • Body - This is where you can put any arbitrary content. Keep it short and simple. Avoid interactive elements, especially in an interactive card.
  • Button row - Space at the bottom for buttons, icon buttons, and links. Button text should use title case.

Accessibility

Screen Reader Support

For clickable cards, use the ariaLabel prop to describe to screen reader users what clicking on the card will do (e.g., Go to medication list, Book an appointment, Read information about type 2 diabetes).

Implementation

Interactive Elements in Interactive Cards

Buttons

Elements that get passed to the buttons prop will be cloned and have events added to them so that:

  • The Card hover state is not triggered when hovering the buttons
  • Clicking on or pressing "Enter" (when focused on the button) will not also trigger the Card onClick

Children

Other interactive elements inside Card (such as a link in text) will not have the same behavior as buttons. Hovering over them will not change the Card hover state, and clicking them will also trigger the Card onClick.

To keep the Card onClick from firing when clicking a link, use the stopPropagation() function before the end of the click handler:

const handleLinkClick = (e) => {
  // do stuff here
  e.stopPropagation();
}

Padding

By default, the padding prop is set to true and adds padding around all content inside Card except for the mediaSlot. Set padding to false to remove it.

Interactivity

Clickable cards show a blue border on hover. They can also have buttons added near the bottom of the card. Other interactive elements within a clickable card (such as links) should be avoided for the design and technical reasons already mentioned.

Card Props

Prop Type Default Description
ariaLabel string Text to tell the screenreader what clicking on the card will do (if onClick is provided)
buttons union
  • node
  • array of nodes
 Single button or array of buttons to be displayed at the bottom of the card — can be forge `Button`, forge `IconButton`, or a custom component
className string Adds a class to the root element of the component
headingDescriptionText string Text to display in the card heading description
headingTag enum
  • 'h1'
  • 'h2'
  • 'h3'
  • 'h4'
  • 'h5'
  • 'h6'
 'h2' Specifies the heading's HTML tag. For more on the importance of heading tags in semantic HTML, see the W3C's article on this topic.
headingText string Text to display in the card heading
mediaSlot node Slot intended for media that will be full width at the top of the card — images default to full width of the card
onClick func Handles click event on card — does not fire when clicking on an interactive element inside card, such as a button
padded bool true Determines whether padding will be added around the content inside card (except for content in the `mediaSlot` — padding is never added there)
...rest Props passed through to the root element of the component
