Usage

A RemovableTag represents a single piece of user input. It displays a value that the user has entered or selected. It can be used on its own or inside an interactive object, such as Multiselect. Multiple tags may be used together, with a separate tag for each value. RemovableTag is commonly used to display filters when searching and filtering content.

Users can remove a tag by clicking the X (close) icon or, on mobile devices, touching anywhere on the tag. Removing the tag may also prompt a refresh of search results. It should not trigger other interactions (e.g., linking to a new page, opening a pop-up).

RemovableTag should not be used for displaying system-generated labels or calling out information about an object. Use StatusTag for this instead.

Design

The tag's width depends on the length of the label text inside it. Labels should be displayed on one row with no wrapping. Label text longer than maxCharacters (about 30 characters by default) is automatically truncated with an ellipsis (...). Users can hover to reveal the full content in a tooltip (title attribute). Do not use icons inside the tag.

For desktop use, the tag's hover target for removal is the X (close) icon. On mobile devices, the touch target is the entire tag.

RemovableTag has two sizes: default and small. The small variation has a smaller height and font size and is useful for displaying long text labels. Default- and small-sized tags should not be mixed together.

The colors for this component should not be changed.

Implementation

Developers must implement the onRemove behavior that removes tags when they are clicked or touched. This functionality is not actually built into RemovableTag.

You do not need to explicitly set the tag as mobile or desktop. The handling for both is built in. It's based on event handling. RemovableTag listens for click events on the X (remove) icon and for touch events on the entire tag.

By default, label text is limited to approximately 30 characters. Longer text is automatically displayed as truncated with an ellipsis (...). Although not generally recommended, this can be overridden in CSS by setting max-width for the fe_c_removable-tag__text element (with CSS specificity of at least two classes).
