Usage

The ReduxForm component is a better choice than the generic Form component if the application using Forge is already built on the Redux architecture. In addition to the state management structure of Redux/Redux Form, it also provides features like validation.

ReduxForm is meant to be used with ReduxFormField.

Design

ReduxForm is simply a wrapper around the Form component with added functionality for managing the component's state. Their designs are completely identical. Design documentation for Form should be consulted when using ReduxForm.

Find more guidance on the Form Design Foundations page.

Implementation

The ReduxForm component is a wrapper around Form. All Form props can be used on ReduxForm.

ReduxForm uses the Redux Form library for full state management of forms. To enable this, the formReducer from the 'redux-form' package must be passed to the application's store using a key called 'form'. This single reducer will serve for all form components application-wide.

import { createStore, combineReducers } from 'redux'
import { reducer as formReducer } from 'redux-form'

const rootReducer = combineReducers({
  // ...your other reducers here
  // you have to pass formReducer under the 'form' key,
  // for custom keys see docs for Redux Form getFormState
  form: formReducer
})

const theStore = createStore(rootReducer)

<Provider store={theStore}>
    <div>
        <ReduxForm form=”this-form-uid” onSubmit={this.handleSubmit}>
        //ReduxFormFields go here
        </ReduxForm>
    </div>
</Provider>

Each ReduxForm has two required properties:

  • form: Used as a unique identifier in the applications store
  • onSubmit: Callback function which receives the form's field values as an object of key-value pairs

Customizing the Button Row

To use a custom Button row instead of the default button:

  1. Set the includeSubmitButton prop to false to turn off the default.
  2. Pass the custom button row component into the inputSlot prop of the FormFieldLayout component. Using FormFieldLayout will keep the button row in alignment with the other FormFields.
    <FormFieldLayout inputSlot={<MyButtonRow />} />

ImmutableJS

Different classes and methods from the 'redux-form' package are required when using ReduxForm with ImmutableJS. To use ReduxForm with ImmutableJS, refer to the Redux-Form ImmutableJS example, create a compatible form as shown below, and set the redux-form's immutable Field object as the reduxFieldAs prop of each ReduxFormField.

import { reduxForm } from 'redux-form/immutable'; // <--- immutable
import { ReduxForm } from '@athena/forge';

const ImmutableForm =
  reduxForm(ReduxForm.REDUX_FORM_ARGS)(ReduxForm.REDUX_FORM_BASE);

export default ImmutableForm;

SubmissionError

When running external validation on form submission, including server-side validation, you can tell ReduxForm that there were errors by throwing a SubmissionError as demonstrated in this example. ReduxForm doesn't currently display the form-level errors associated with the _error key, but does display field-specific errors. If the errors aren't displayed, it could be because of an inconsistency in how SubmissionError is imported. This only happens if you're calling the submit method directly. ReduxForm Props

In addition to the listed props, ReduxForm accepts all Form props.

Prop Type Default Description
form (required) string A string that represents the unique id of form in the redux store
onSubmit (required) func A callback function that is invoked when the form is submitting and receives the following params:
  • values (Object): The field values in the form of { field1: 'value1', field2: 'value2' }.
  • dispatch (Function): The Redux dispatch function.
  • props (Object): The props passed into your decorated component.
asyncBlurFields array Field names for which onBlur should trigger a call to the asyncValidate function
asyncValidate func A function that takes all the form values, the dispatch function, the props given to your component and the current blurred field, and returns a Promise that will resolve if the validation is passed, or will reject with an object of validation errors in the form
buttonText string Text used on the submit button
includeSubmitButton bool true Whether the default submit button should be used
initialValues object The values with which to initialize your form in componentWillMount(). The values should be in the form { field1: 'value1', field2: 'value2' }.
onChange func A callback function that will be called with all the form values any time any of the form values change. onChange will be called with the following parameters:
  • values (Object): The field values in the form of { field1: 'value1', field2: 'value2' }.
  • dispatch (Function): The Redux dispatch function.
  • props (Object): The props passed into your decorated component.
onSubmitFail func A callback function that will be called when a submission fails for whatever reason. It will be called with the following parameters:
  • errors (Object): the errors that caused the submission to fail.
  • dispatch (Function): The Redux dispatch function.

  • props (Object): The props passed into your decorated component.
onSubmitSuccess func A callback function that will be called when a submission succeeds. It will be called with the following parameters:
  • Result (Object): Any result that onSubmit has returned.
  • dispatch (Function): The Redux dispatch function.
  • props (Object): The props passed into your decorated component.
validate func A synchronous validation function that takes the form values and props passed into your component. If validation passes, it should return {}. If validation fails, it should return the validation errors in the form { field1: <String>, field2: <String> }. Defaults to (values, props) => ({}).
