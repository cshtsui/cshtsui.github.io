Multiselect

Multiselect allows users to choose one or more options from a dropdown list. Users can type or check off the options.

Usage

Multiselect is useful when presenting a list of five or more options. It's similar to SelectionList, but the option list is hidden by default. This can allow a huge list of options without taking up lots of space on the page.

Selecting and deselecting options

Users can select options by checking them off or by typing in the selection field. Selected options are appended to a list inside the selection field. To deselect options, users can uncheck them from the option list, click the "x" next to them in the selection field, or delete/backspace in the selection field like deleting text.

Filtering the option list

Typing in the selection field automatically filters the option list, matching options with the input string anywhere in their text. This is useful for finding specific options in lists (e.g., typing "hyper" to narrow down a large list of diagnoses).

If the string doesn't match any options, the option list displays the noOptionsMessage prop, which defaults to "No options found".

Design

Selection field height

The selection field's height will expand if the list of selected options runs past one line. This may affect your layout. The height stops expanding at a max of four lines of text. When that is exceeded, a scroll bar will automatically display on the selection field for viewing all of the selected options.

Options

User can click anywhere on an option to select or deselect it: checkbox, option text, and any whitespace on the same line. Make your options text only. Do not put clickable elements inside options.

There is no max length for option text. When selected and added to the selection list, a long text option is automatically truncated to one line of text.

Custom text

The selection field's default placeholder text, "Type or select item(s)", should be customized to reflect the context. For example, "Type or select patient(s)", "Type or select medication(s)".

The filtering message "No options found" can be customized with the noOptionsMessage prop.

Form field label text should use sentence case ("Patient name", not "Patient Name"). Do not use the placeholder text instead of a label.

Find more guidance on the Form Design Foundations page.

Implementation

Multiselect can be used on its own or as an input in FormField or ReduxFormField.

Asynchronous options

Multiselect is meant to be used synchronously. If you would like to load options asynchronously, you can use the MultiselectAsync extension.

Value structure

Multiselect takes as its value an object mapping of option values to boolean values. If there are three items A, B, and C and you want only B checked, set value (or defaultValue if you are using Multiselect as an uncontrolled component) to

{
  'A': false,
  'B': true,
  'C': false,
}

Alternatively, only selected values are required. You can represent the same value structure with

{
  'B': true,
}
