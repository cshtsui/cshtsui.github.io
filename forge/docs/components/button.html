Usage

Buttons are utilized for actions that will advance users through the interface and their tasks. They require users to make a specific choice before moving forward with their associated action.

Buttons are commonly used to complete or advance workflows (e.g., adding medication to a patient’s chart). They are also frequently used to commit transactions (e.g., signing an order).

Buttons vs. Links

Links are typically preferred for navigation to another page within the application. See the Typography Foundations page for more information on links.

Design

Button text should be concise (one or two words), start with action words, and use title case (e.g., "Confirm Appointment", not "Confirm appointment").

If button text is long, then it may be trying to explain too much about its function. Consider clarifying this somewhere else on the page.

When a primary button is paired with a secondary button, they always appear side-by-side. For small viewports, they may be stacked vertically.

If you are using 'Button' as part of a form, you can find more guidance on the Form Design Foundations page.

Primary Button

Primary buttons present an important ("primary") call to action. They are used to drive the user to take action, and are directly responsible for form or workflow completion, such as save, continue, or submit. Where possible, align primary actions with input fields for a clear path of completion.

More than one primary button can be used per page or section, but two primary buttons should never be used next to each other, as they would compete for importance.

Secondary Button

Secondary buttons are used for less prominent actions such as back, reset, or cancel. They can accompany primary buttons or appear on their own in a secondary section of a page. Multiple secondary buttons can be placed next to each other if their actions are of equal value to the user.

Disabled Button

Use a disabled button when a user should not be able to complete the action without fulfilling some other requirement on the screen.

Full-Width Button

A button may be made to take up all space in its parent container rather than take the length of its label copy. This is particularly useful when creating layouts based on Grid and the button's width should be fluid.

Sizing for Buttons

The standard size button was designed to be used with the typical sizes of the Forge form components and should be the first choice for buttons. For exceptional cases, three other sizes that are multiples of the standard size are provided. Sizing affects both the font-size and padding on the button, resulting in impacts to the height and width (which is still primarily determined by label copy length).

Avoid setting arbitrary sizes of buttons with fixed pixel dimensions. Instead, consider using Grid and a full-width button if different button dimensions are needed.

Icon in a Button

An icon can be added for additional visual emphasis by using the icon prop. See Icon for a list of available icons. Button text is still required even if it has an icon, but otherwise the guidance is the same as for IconButton.

Icons are available only in standard-size buttons. They can't be used in tiny, small, or large buttons.

Referring to Buttons in Page Copy

The button name appears in bold and title case.

  • Do this: "Enter your password and click Log In."
  • Not this: "Enter your password and click log in."

Omit the word "button" unless doing so creates confusion.

  • Do this: "Click Log In"
  • Not this: "Click the Log In button"

Indicate action by asking users to "click" a button.

  • Do this: "Click Log In."
  • Not this: "Press Log In." or "Hit Log In."
  • Exception: In touch screen interfaces, "tap" is appropriate instead of "click."

Button Props

Prop Type Default Description
text (required) string Button text
className string Adds a class to the root element of the component
disabled bool false Applies the disabled state class and adds disabled attribute to the button element
fullWidth bool false Applies the full width class to make button the full width of its parent
icon string The icon to display inside the button. Only applicable to default size. Note it is not recommended to use the solid, special icons (Attention, Critical, Info, and Success) within Button
iconSet objectOf
  • IconBase components
 ForgeIcons Specify a custom icon set defined as a dictionary of IconBase components. Icon sets can be generated using the App Fabric Forge Extension Generator.
onClick func () => {} Handles click event on the button
sizing custom: ForgePropTypes.sizes When specified, this can change button size from normal to tiny, small, or large
type string 'button' Sets the type attribute on the button element (e.g., 'button', 'submit', or 'reset')
variant enum
  • 'primary'
  • 'secondary'
 'primary' Applies the primary or secondary button class
...rest Passthrough props to the <button>
