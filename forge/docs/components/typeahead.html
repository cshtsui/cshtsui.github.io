Typeahead

Typeahead is a text input that shows autocomplete-style suggestions as the user types, accelerating the user's input process.

Usage

Typeahead is a text input field with autocomplete behavior. When users begin typing, it shows a dropdown list of suggested values. Users can then click or arrow down to a value instead of having to type the entire string. This helps speed up the user's input process.

Typeahead is useful when users are likely to enter a value from a known list, such as diagnoses or medications.

Typeahead gets its suggestions by string matching the user's input against a reference list. See Design for default logic and ways to customize this.

Users can also enter a free text value that is not on the reference list. If you want to force users to choose from a predefined list, use Select or Multiselect instead.

Design

String matching settings

Typeahead's string matching behavior has settings that can be customized:

  • Matching string location in list items

    • Default: Match list items that contain the input string anywhere in the text
    • Customize it: Change this to match only items that begin with the input string by setting the matchPos prop to start

  • Case/capitalization

    • Default: Case insensitive (for example, the string "Alzheimer's" will also match "alzheimer's")
    • Customize it: Change this to case sensitive by setting the ignoreCase prop to false

  • Length of text users must type before suggestions appear

    • Default: One character
    • Customize it: Change this by setting the minQueryLength prop

Text input field

The input field may optionally display an Icon within the left side of the field. This icon should visually reinforce the action of the field (e.g., use the Filter icon when entries in the input filter a nearby list). Use the icon prop to customize this.

The input field's default placeholder text should be customized to reflect the context (e.g., "Enter a diagnosis"). Use the placeholder prop to set this.

Find more guidance on the Form Design Foundations page.

Accessibility

Screen Reader Support

Available suggestions are announced to a screen reader when they appear on the screen. If you are using Typeahead as a controlled component, you need to provide the suggestionsAnnouncement prop to expose available suggestions to screen readers.

Implementation

The Typeahead component is based on React Autosuggest. Forge leverages third-party libraries when possible, but they must still achieve Forge design standards and present a unified API. Only the functionality described in the Forge documentation is officially supported for this component, even if an underlying third-party API currently remains usable.

Options and Suggestions

Typeahead expects options or suggestions to be passed as an array of either strings or objects with a value property. options is the set of all possible values to display in the dropdown and leaves the filtering to internal logic. suggestions is the set of values to display currently and should already have been filtered. Set one or the other, not both.

List Display

Both the options and suggestions properties populate the list below the input. For uncontrolled components use options and for controlled components use suggestions.

If the suggestions property is used, the list will be displayed immediately unless the alwaysRenderSuggestions is set to false.

Use the minQueryLength property in conjunction with options to display the list only after a certain number of characters has been entered.

Updating suggestions will update the dropdown while open, but updating options will not.

Filtering

By default in uncontrolled mode Typeahead does a string match between the entered text and the options. If a different method of filtering is desired, a custom function can be passed to the filterOptions prop that will allow for different modes of filtering.

Excluded Props

Typeahead does not make use of some of the properties from React Autosuggest mostly related to having sections within the suggestions list: multiSection, renderSectionTitle, getSectionSuggestions, getSuggestionValue.

Async suggestions

Typeahead expects suggestions to be set synchronously when onSuggestionsFetchRequested or onSuggestionsClearRequested is called. There is no option for loading it asynchronously, at the moment. This is on the Forge team's list for possible future implementation.
