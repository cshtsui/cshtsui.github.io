DateInput

DateInput lets users enter a date by typing or by selecting in a popup calendar.

            <li>Date value: {this.state.date && DateInput.formatDate(this.state.date)}</li>
          </ul>
        </GridCol>
      </GridRow>
    );
  }
}

Usage

Use DateInput when users need to provide date information, such as date of birth. It can handle either manual entry or selection from a calendar interface.

Keyboard Shortcuts

The DateInput component supports a variety of shortcuts and relative formatting functionality:

  • Typing t will display today's date
  • Typing t+[# of days] to display a future date
  • Typing t-[# of days] to display a past date

Feedback appears below the DateInput when a user begins to type a shortcut if DateInput is within a FormField or ReduxFormField. To hook into these messages outside of a FormField, use the onShortcutChange prop or the onHintTextChange prop.

Design

By default, the popup calendar opens when a user clicks into the DateInput field. This is helpful when users are selecting dates close to the present time.

If you focus into the DateInput field by using the Tab key, this behavior changes. The popup calendar will not open immediately, but you can still access it by clicking the input or the calendar icon or by pressing Space or Enter when the calendar icon is focused. We do this so it is easier to quickly move past the DateInput field when using the keyboard for navigation.

When a required date entry is distant from the present time, popping the calendar on click can be disabled by setting the showCalendar prop to false. Users can still access the popup calendar by clicking on the calendar icon if they prefer date selection from the calendar.

If the current year is outside of the range set by the selectYearStart and selectYearEnd props the calendar will open to January 1st of nearest valid year.

Label text should use sentence case (e.g., "Date of birth").

Find more guidance on the Form Design Foundations page.

Accessibility

Although the DateInput input field is accessible as a standard text input, the popup calendar is not currently accessible to keyboard-only users or screen readers. We plan to address this in a future release.

Implementation

DateInput is rendered as an input[type="text"] to avoid the browser's default date input UI.

DateInput Props

Prop Type Default Description
className string Adds classes to the root of DateInput.
dayModifiers object Applies modifier classes to sets of days. All dates should be created with moment.utc. See DayPicker API for more details.
dayPickerProps object Passes through to DayPickerInput. The following dayPickerProps are defined and cannot be overridden: captionElement, navbarElement. Use at your own risk and note that DayPicker assumes dates are midnight local time, not UTC.
disabled bool false Applies styles to the input in addition to preventing changes
disabledDays
  • UTC Moment object
  • Range object
  • Before/After object
  • daysOfWeek object
  • Function taking the day (local JS Date) and returning a bool
  • Array of any combination of the above
Disables specified days in the calendar. All dates should be created with moment.utc. Note that the calendar doesn't prevent viewing months whose days are all disabled.

Examples of special types:
  • Range object
    { from: moment.utc(), to: moment.utc('01-01-2020') }
  • Before/After object
    {before: moment.utc(), after: moment.utc('01-01-2021')}
  • daysOfWeek object
    {daysOfWeek: [0,6]}
  • Function
    (day) => day.getDay() === 0
error string Highlights the input when defined (specific string doesn't matter)
hintText string Adjusts calendar position to expose hint text when defined (specific string doesn't matter)
htmlRef func ref function for getting a reference to the underlying input
isKeyboardUser bool Notes whether the user is in "keyboard" mode. Prevents the calendar from opening on focus if true. Set automatically if a child of Root.
onBlur func Triggered when the DateInput component loses focus. Not triggered when focus shifts between the input and the calendar.
onCalendarMonthChange func Triggered when the user changes which month (or year) is visible in the calendar. Intended for cases where calendar data like disabledDays is loaded incrementally.
onChange func Triggered when the value changes from the user typing, using a shortcut, or choosing a day in the calendar. Provides the formatted value as event.target.value and the unformatted value as event.target.rawValue.
onDateChange func Triggered after change or blur when current value is a valid date. Adds parsedDate property of type moment to event.target.
onErrorChange func Handles the input reporting a change in its error status or message
onHintTextChange func Handles the input reporting a change in its hint text message
onShortcutChange func Triggered after keypress while inputting a shortcut. The shortcut status is available as event.target.shortcutState.
selectYearEnd number date().year() + 3 The upper bound of a range used to customize the calendar picker's year selection. Does not impact prev/next month navigation.
selectYearStart number date().year() - 3 The lower bound of a range used to customize the calendar picker's year selection. Does not impact prev/next month navigation.
showCalendar bool true When true calendar picker is shown automatically on input focus
theme custom: ThemeConsumer.themePropType Theme variables for configuring an instance of Forge. See Root component for more details.
...rest Passthrough props to DayPickerInput's inputProps.

Date parsing/formatting API

Class Field/Method Type Description
DateInput.DATE_FORMAT string Constant that represents the date format: 'MM-DD-YYYY'
DateInput.DATE_FORMAT_CONSTRAINT object Constraint object containing the regex 'pattern' and 'message' used by DateInput
DateInput.formatDate(Date) func Returns a formatted string representing the date.
DateInput.parseDate(string) func Takes a well-formatted string and returns a Date object. If the string isn't in DATE_FORMAT, it returns an invalid date object.
