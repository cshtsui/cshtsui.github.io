Usage

Root should be placed at the top of a page or section of a page that uses Forge styling or components. The component itself is not visible to the user.

Implementation

Root is required at the head of any section of a React application that will include Forge components. It has two functions: providing a shared theme and adding CSS classes.

Theme

Root uses the Context API to share a theme object with any components that need it. If you customize your Sass using a theme file, you can use this theme object to expose the same customization to React code. This theme only includes fields that components need access to in React. Currently, the only field is rootClasses.

As an alternative to using Root to provide the theme context, you can use ThemeContext directly. The syntax is <ThemeContext.Provider value={theme} >.

To write a component that has access to the theme, give it a prop named theme and wrap it in the ThemeConsumer Higher-Order Component (HOC). The HOC will supply the theme context to the prop to be used as any other prop (including lifecycle methods).

CSS Classes

Root renders a div with the fe_f_all class to ensure that all content and components under it render with correct Forge styles. It also adds any classes listed in rootClasses in the theme. If you need to, you can add these classes to the DOM yourself without using Root.

Form Props

Prop Type Default Description
className string Adds a class to the root element of the component
theme custom: ThemeContext.Provider.propTypes.value shape:
  • rootClasses: string or arrayOf string
 Theme variables for configuring an instance of Forge. Currently includes:
  • rootClasses - class or array of classes to add to root DOM elements (in addition to fe_f_all)
...rest All other props are passed to the root element.

Class Applies to Description
