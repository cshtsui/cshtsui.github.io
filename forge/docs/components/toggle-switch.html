ToggleSwitch

This selector lets users choose between 2 opposing states, such as "on" and "off". When it is set, the change is saved instantly.

You must import this component separately.
It is not part of Forge Core yet. It may undergo further design/API changes. See the Extensions page for more info.

Usage

ToggleSwitch is a selector that lets users choose between 2 opposing states. It's similar to a light switch with its states of "on" and "off". The component includes a switch and text descriptions of the states. A label (or descriptive text that acts as a label) is typically used with this component.

ToggleSwitch is often used for editing user preferences and system settings. When users change the switch's state, the change is saved instantly with no need to click a separate Save button.

ToggleSwitch may seem similar to Checkbox. Use these guidelines if you are unclear on which component to use:

  • Use ToggleSwitch if:
    • The changes should take effect immediately.
    • The user doesn't need to review or confirm the selection.
    • There are only 2 possible values for the setting, not more.
    • There is a default option.
  • Use Checkbox if:
    • The user needs to take an extra step before the changes take effect, such as reviewing the selection or clicking a Save button.
    • There are more than 2 possible values for the setting.
    • All of the possible options need to be displayed. (ToggleSwitch shows only the current state, not all possible states.)

The entire switch (plus any state description text) is a target. If the user clicks or taps anywhere on the switch, its value is reset to the opposite state.

Design

Sizes

ToggleSwitch is available in two sizes: Desktop (default size) and Mobile (larger size).

Label

ToggleSwitch should be used together with copy that serves as a label, explaining what the switch controls. This copy should be placed to the left of the switch.

The ToggleSwitch component itself does not include a label. Use the component with FormField or ReduxFormField to include a form-style label, or add your own copy next to the standalone component. Formatting may vary; the copy is not required to be styled like a form field label.

This copy may describe an action ("Show billing codes", "Include patient details") or may simply be the name of a setting that is turned on or off ("Email notifications"). A quick way to check whether your label is clear is to read it aloud and then add "On/Off" (or your state descriptors, if they are different). If it doesn't make sense, consider rewriting the copy (credit to Nielsen Norman Group's toggle-switch guidelines).

State Descriptors (optional)

State descriptor copy is displayed to the right of the switch. Only the selected state is shown.

Displaying state descriptor copy is optional. It may be omitted if the states are essentially just versions of "on/off" (e.g., active/inactive, enabled/disabled). In this case, the switch itself can visually convey whether it's set to "on" or "off". State descriptor copy should be displayed if the states are more meaningful than "on/off" (e.g., "show/hide") or if displaying the copy will help with clarity.

State descriptor copy should be pairs of opposite descriptions, such as:

  • On / Off
  • Active / Inactive
  • Enabled / Disabled
  • Show / Hide

They should not be instructions about what to do to the switch or what will happen if it's reset. Think adjectives, not verbs. State descriptors should be short and use sentence text.

Accessibility

ToggleSwitch includes several built-in features that support accessibility, including:

  • Keyboard-only use
    • When the switch is focused, users can toggle it by pressing the enter or space key.
  • Screen reader support
    • Screen readers will announce the switch's current setting (on/off) and, if displayed, the selected state descriptor text.

Implementation

ToggleSwitch can be used on its own or within a FormField or ReduxFormField for features like field labels and error messaging.

Under the hood, this component uses a HTML button with the "switch" aria role. The button contains a div used to style the switch position and, if the descriptions prop is used, will also contain a dynamic HTML label that changes value to match the switch state.

Controlled vs. Uncontrolled

To use ToggleSwitch as a controlled component, set the checked prop. To set a default toggle state but allow it to control its own state, use the defaultChecked prop instead.
