Usage

The Input component is commonly used in forms. FormField and ReduxFormField provide a responsive layout for inputs being used in forms. Additionally, they bundle a given input with a label, a component for error messages, and a component for informational (hint) text. Use Input with Form and FormField (or their Redux versions) whenever possible to promote a consistent look and feel for users as well as benefit from conveniences for commonly needed form functionality.

Use the ReadOnlyInput component when displaying information in a form that can never be altered by the user. Do not use a disabled Input for this purpose, as it can imply that an action in the workflow might enable editing. Controls that are disabled also have lower contrast and hinder readability.

Input type

Input can be optimized for different types of input. The allowed types are text, number, email, password, search, tel, and url. Some of these types have a visual or functional impact in browsers. For example, a password input's value is obscured for improved security. See the demos for examples and be aware that it differs across browsers and between desktop and mobile.

Design

Input may optionally display any Icon within the left side of the field. These should be used to reinforce the action of the field (e.g., using the Filter icon when entries in the input filter a nearby list).

Label text should use sentence case ("Patient name", not "Patient Name").

Do not use an input's placeholder text instead of a label. This affects accessibility. Labels provide context. When users click into an input field, placeholder text disappears. This is not helpful for users with reading or memory issues who may need a persistent reminder of what the input field is for.

Find more guidance on the Form Design Foundations page.

Input Props

Prop Type Default Description
className string Adds a class to the root element of the component
disabled bool false Adds the relevant Forge state class to the input and passes prop to native react <input />
error string Highlights the input when defined (specific string doesn't matter)
inputWidth string Styling to be applied to width css property. Specifying this will override all other behavior that affects input width.
icon string Which icon to display inside the field
iconSet objectOf
  • IconBase components
 ForgeIcons Specify a custom icon set defined as a dictionary of IconBase components. Icon sets can be generated using the App Fabric Forge Extension Generator.
type enum
  • 'text'
  • 'number'
  • 'email'
  • 'password'
  • 'search'
  • 'tel'
  • 'url'
 'text' Type of text entry input to display
...rest Passthrough props to the input element
